Description

Boost.SIMD functions and constants are designed to work seamlessly on both scalar and SIMD parameters.

Their polymorphic behavior ensure that the best available implementation will be selected for any given type or types combination.

Boost.SIMD functions are accessible via its eponymous include file located in the boost/simd/function folder. E.g, boost::simd::abs requires the inclusion of boost/simd/function/abs.hpp.

Similarly, Boost.SIMD constants are accessible via its eponymous include file located in the boost/simd/constant folder. E.g, boost::simd::True requires the inclusion of boost/simd/constant/true.hpp. Note that contrary to functions, all Boost.SIMD constants name starts with a capital letter.

Function semantics are usually similar to the pre-existing ones, if any, with the following caveats:

  • All functions are assumed to be noexcept.
  • Type mixing may be prohibited for fundamental types , i.e boost::simd::plus (char,char) is valid while boost::simd::plus (int,char) is not
  • Type promotion on result type may be disabled, i.e boost::simd::plus (char,char) returns char

Those changes are motivated by the fact SIMD operations require similar limitations.
